# Deep Dive

Long-form documentation of how the system actually works — architecture, per-flow walkthroughs, end-to-end scenarios, postmortems for non-trivial bugs.

## When to add a doc here

- A subsystem or flow needs explanation that's too long for a feature spec.
- A bug investigation produced findings worth keeping (postmortem).
- A new contributor needs a "how does X actually work" reference.

## Conventions

- File names: `<topic>.md` for evergreen explanations, `YYYY-MM-DD-<slug>.md` for time-anchored postmortems.
- Index this folder in its own README as it grows.
