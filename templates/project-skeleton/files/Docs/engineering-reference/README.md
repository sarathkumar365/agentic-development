# Engineering Reference

Evergreen internal reference material — things you look up while working, not feature-specific.

## What goes here

- Library/framework migration plans
- Testing conventions and how-tos
- Known issues lists that span features
- Language, tooling, or infra cheat-sheets

## What does not go here

- Feature specs → `../features/`
- One-off investigations → `../deep-dive/`
- Decisions → `../repo-decisions/`
