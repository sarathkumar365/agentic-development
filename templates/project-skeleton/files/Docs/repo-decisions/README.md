# Repository Decisions

Repo-wide architectural and process decisions (ADRs) that apply beyond a single feature.

## Mandatory read order (before implementation)

1. This file
2. All `Accepted` decision files relevant to the modules you're touching
3. The relevant feature docs under `../features/<feature-slug>/`

## Decision lifecycle

- Status values: `Proposed`, `Accepted`, `Superseded`, `Deprecated`
- Implementation-critical behavior must reference `Accepted` decisions.
- If a feature RFC introduces a repo-wide decision, promote it to this folder in the same phase.
